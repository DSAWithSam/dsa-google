# dsa-google

Google-specific DSA preparation - Tutorials focused on understanding, not memorization.

This repository contains carefully selected problems that frequently appear in Google interviews, organized by difficulty (Easy, Medium, Hard). 

Each tutorial follows a practical flow: problem statement, clarifying questions, pseudocode, brute force, optimal solution, complexity analysis, common mistakes, and real interview follow-ups.

### Why These Problems?

These are not random LeetCode problems. They are chosen because:
- They appear often in actual Google interviews
- They teach important patterns that transfer to many other questions
- They build on each other progressively

### Repository Structure

- **easy/** → Foundational problems that build core skills
- **medium/** → High-ROI problems that form the bulk of Google interviews  
- **hard/** → Senior-level problems that separate strong candidates
- **patterns.md** → Google-favorite patterns and templates
- **roadmap.md** → Suggested learning order

### How to Use This Repo

Start with the **Easy Track** in order.  
Read the tutorial, understand the intuition, implement the solutions yourself, then try the follow-ups.

All code examples are in Python for readability and interview speed (Note: Some of the solutions will not work if you were to copy them directly into LeetCode. The purpose of this tutorial series is to teach you how to tackle these problems, not provide you with 1:1 LeetCode solutions. The code will be functional if you were to plug it into a Python interpreter, and test cases will be included to match. I prefer to use: online-python.com as my online Python interpreter).
